### <a name="pscmdletthrowterminatingerror-inside-trycatch"></a><span data-ttu-id="51971-264">$PSCmdlet.ThrowTerminatingError() inside try/catch</span><span class="sxs-lookup"><span data-stu-id="51971-264">$PSCmdlet.ThrowTerminatingError() inside try/catch</span></span>

### <a name="public-function-templates"></a><span data-ttu-id="51971-268">公共函数模板</span><span class="sxs-lookup"><span data-stu-id="51971-268">Public function templates</span></span>

<span data-ttu-id="51971-269">我与 Kirk Munro 谈论的最后一个要点是，他在所有高级函数的每个 `begin`、`process` 和 `end` 块周围放置了 `try{...}catch{...}`。</span><span class="sxs-lookup"><span data-stu-id="51971-269">One last take a way I had with my conversation with Kirk Munro was that he places a `try{...}catch{...}` around every `begin`, `process` and `end` block in all of his advanced functions.</span></span> <span data-ttu-id="51971-270">在这些泛型 catch 块中，他使用 `$PSCmdlet.ThrowTerminatingError($PSitem)` 作为单行处理所有离开函数的异常。</span><span class="sxs-lookup"><span data-stu-id="51971-270">In those generic catch blocks, he as a single line using `$PSCmdlet.ThrowTerminatingError($PSitem)` to deal with all exceptions leaving his functions.</span></span>

```powershell
    param()
```

## <a name="trap"></a><span data-ttu-id="51971-273">Trap</span><span class="sxs-lookup"><span data-stu-id="51971-273">Trap</span></span>

<span data-ttu-id="51971-274">我将重点放在了异常的 `try/catch` 方面。</span><span class="sxs-lookup"><span data-stu-id="51971-274">I focused on the `try/catch` aspect of exceptions.</span></span> <span data-ttu-id="51971-275">但在结束之前，我需要提到一项旧功能。</span><span class="sxs-lookup"><span data-stu-id="51971-275">But there's one legacy feature I need to mention before we wrap this up.</span></span>

<span data-ttu-id="51971-276">将 `trap` 放置在脚本或函数中，以捕获在该作用域内发生的所有异常。</span><span class="sxs-lookup"><span data-stu-id="51971-276">A `trap` is placed in a script or function to catch all exceptions that happen in that scope.</span></span> <span data-ttu-id="51971-277">发生异常时，将执行 `trap` 中的代码，然后继续执行正常代码。</span><span class="sxs-lookup"><span data-stu-id="51971-277">When an exception happens, the code in the `trap` is executed and then the normal code continues.</span></span> <span data-ttu-id="51971-278">如果发生多个异常，则会反复调用 trap。</span><span class="sxs-lookup"><span data-stu-id="51971-278">If multiple exceptions happen, then the trap is called over and over.</span></span>


<span data-ttu-id="51971-279">我个人从未采用这种方法，但我会在管理或控制器脚本中看到用于记录任何和所有异常的值，然后仍继续执行。</span><span class="sxs-lookup"><span data-stu-id="51971-279">I personally never adopted this approach but I can see the value in admin or controller scripts that log any and all exceptions, then still continue to execute.</span></span>
