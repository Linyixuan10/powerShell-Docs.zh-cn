## <a name="creating-resource-properties"></a><span data-ttu-id="1b144-110">创建资源属性</span><span class="sxs-lookup"><span data-stu-id="1b144-110">Creating resource properties</span></span>
<span data-ttu-id="1b144-111">我们首先要确定资源将公开的属性。</span><span class="sxs-lookup"><span data-stu-id="1b144-111">The first thing we need to do is decide on properties that the resource will expose.</span></span> <span data-ttu-id="1b144-112">在此示例中，我们将通过以下属性来定义 Active Directory 用户。</span><span class="sxs-lookup"><span data-stu-id="1b144-112">For this example, we will define an Active Directory user with the following properties.</span></span>

<span data-ttu-id="1b144-113">参数名称  说明</span><span class="sxs-lookup"><span data-stu-id="1b144-113">Parameter name  Description</span></span>

* <span data-ttu-id="1b144-114">**UserName**：唯一标识用户的键属性。</span><span class="sxs-lookup"><span data-stu-id="1b144-114">**UserName**: Key property that uniquely identifies a user.</span></span>
* <span data-ttu-id="1b144-115">**Ensure**：指定用户帐户应该为 Present 还是 Absent。</span><span class="sxs-lookup"><span data-stu-id="1b144-115">**Ensure**: Specifies whether the user account should be Present or Absent.</span></span> <span data-ttu-id="1b144-116">此参数只有两个可能的值。</span><span class="sxs-lookup"><span data-stu-id="1b144-116">This parameter will have only two possible values.</span></span>
* <span data-ttu-id="1b144-117">**DomainCredential**：用户的域密码。</span><span class="sxs-lookup"><span data-stu-id="1b144-117">**DomainCredential**: The domain password for the user.</span></span>
* <span data-ttu-id="1b144-118">**Password**：允许必要时配置对用户密码进行更改所需的用户密码。</span><span class="sxs-lookup"><span data-stu-id="1b144-118">**Password**: The desired password for the user to allow a configuration to change the user password if necessary.</span></span>

<span data-ttu-id="1b144-119">我们使用 **New-xDscResourceProperty** cmdlet 来创建属性。</span><span class="sxs-lookup"><span data-stu-id="1b144-119">To create the properties, we use the **New-xDscResourceProperty** cmdlet.</span></span> <span data-ttu-id="1b144-120">下面的 PowerShell 命令可以创建上述属性。</span><span class="sxs-lookup"><span data-stu-id="1b144-120">The following PowerShell commands create the properties described above.</span></span>

```powershell
$UserName = New-xDscResourceProperty –Name UserName -Type String -Attribute Key
$Ensure = New-xDscResourceProperty –Name Ensure -Type String -Attribute Write –ValidateSet "Present", "Absent"
$DomainCredential = New-xDscResourceProperty –Name DomainCredential -Type PSCredential -Attribute Write
$Password = New-xDscResourceProperty –Name Password -Type PSCredential -Attribute Write
```

## <a name="create-the-resource"></a><span data-ttu-id="1b144-121">创建资源</span><span class="sxs-lookup"><span data-stu-id="1b144-121">Create the resource</span></span>

<span data-ttu-id="1b144-122">创建好资源属性后，我们就可以调用 **New-xDscResource** cmdlet 来创建资源。</span><span class="sxs-lookup"><span data-stu-id="1b144-122">Now that the resource properties have been created, we can call the **New-xDscResource** cmdlet to create the resource.</span></span> <span data-ttu-id="1b144-123">**New-xDscResource** cmdlet 采用了属性列表作为参数。</span><span class="sxs-lookup"><span data-stu-id="1b144-123">The **New-xDscResource** cmdlet takes the list of properties as parameters.</span></span> <span data-ttu-id="1b144-124">它还采用了应将模块创建于的路径、新资源的名称以及包含它的模块名称。</span><span class="sxs-lookup"><span data-stu-id="1b144-124">It also takes the path where the module should be created, the name of the new resource, and the name of the module in which it is contained.</span></span> <span data-ttu-id="1b144-125">下面的 PowerShell 命令可以创建资源。</span><span class="sxs-lookup"><span data-stu-id="1b144-125">The following PowerShell command creates the resource.</span></span>

<span data-ttu-id="1b144-126">**New-xDscResource** cmdlet 创建 MOF 架构、主干资源脚本、新资源所需目录结构以及公开新资源的模块清单。</span><span class="sxs-lookup"><span data-stu-id="1b144-126">The **New-xDscResource** cmdlet creates the MOF schema, a skeleton resource script, the required directory structure for your new resource, and a manifest for the module that exposes the new resource.</span></span>

<span data-ttu-id="1b144-127">MOF 架构文件位于 **C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.schema.mof**，其内容如下。</span><span class="sxs-lookup"><span data-stu-id="1b144-127">The MOF schema file is at **C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.schema.mof**, and its contents are as follows.</span></span>

```

{
  [Key] string UserName;
  [Write, ValueMap{"Present","Absent"}, Values{"Present","Absent"}] string Ensure;
  [Write, EmbeddedInstance("MSFT_Credential")] String DomainCredential;
  [Write, EmbeddedInstance("MSFT_Credential")] String Password;
};
```

## <a name="updating-the-resource"></a><span data-ttu-id="1b144-131">更新资源</span><span class="sxs-lookup"><span data-stu-id="1b144-131">Updating the resource</span></span>

<span data-ttu-id="1b144-132">如果需要添加或修改资源的参数列表，可以调用 **Update-xDscResource** cmdlet。</span><span class="sxs-lookup"><span data-stu-id="1b144-132">If you need to add or modify the parameter list of the resource, you can call the **Update-xDscResource** cmdlet.</span></span> <span data-ttu-id="1b144-133">该 cmdlet 将使用新参数列表更新资源。</span><span class="sxs-lookup"><span data-stu-id="1b144-133">The cmdlet updates the resource with a new parameter list.</span></span> <span data-ttu-id="1b144-134">如果你已在资源脚本中添加逻辑，它将保持不变。</span><span class="sxs-lookup"><span data-stu-id="1b144-134">If you have already added logic in your resource script, it is left intact.</span></span>

<span data-ttu-id="1b144-135">例如，假如你想为用户将最新日志包含在资源中。</span><span class="sxs-lookup"><span data-stu-id="1b144-135">For example, suppose you want to include the last log in time for the user in our resource.</span></span> <span data-ttu-id="1b144-136">你可以调用 **New-xDscResourceProperty** 创建新属性，然后调用 **Update-xDscResource** 并将新属性添加到属性列表，而不用重新编写资源。</span><span class="sxs-lookup"><span data-stu-id="1b144-136">Rather than writing the resource again completely, you can call the **New-xDscResourceProperty** to create the new property, and then call **Update-xDscResource** and add your new property to the properties list.</span></span>

```powershell

Update-xDscResource –Name 'Demo_ADUser' –Property $UserName, $Ensure, $DomainCredential, $Password, $lastLogon -Force
```
